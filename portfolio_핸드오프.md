> **이 문서를 받은 AI에게:** StarLink 프로젝트의 **포트폴리오 웹페이지(`portfolio.html`)** 초안이 만들어졌습니다.

## 1. 프로젝트 한 줄 요약
**StarLink** — 무선 이어폰(AirPods·갤럭시버즈·MFi 보청기)용 iOS 실시간 동시통역 앱.
주변 영어·일본어·중국어 음성을 **1초 미만 지연**으로 한국어 음성+자막으로 변환.
스택: **Swift/SwiftUI(iOS17+) · Python(livekit-agents) · LiveKit WebRTC · Gemini 3.5 Live Translate**.
규모: Swift ~2,483줄 / Python ~857줄. GitHub: https://github.com/ljhljh0703-cmd/StarLink

## 2. 포트폴리오 페이지 구성 (현재 완성 상태)
라이트 테마 · 한국어 · 세로 스크롤 1페이지. 섹션 순서:

1. **Hero** — 타이틀 + 애니메이션 폰 목업(실시간 자막 데모)
2. **통계 스트립** — `<1s 지연` / `EN·JA·ZH→KO` / `0ms 언어감지` / `3.3K+ 코드라인`
3. **핵심 기능 6** — 카드 그리드
4. **4대 기술 난제** ⭐ — 문제↔해결 대비 구조 + 실제 `detect_language` 코드 스니펫
   - ① Bluetooth/MFi 오디오 라우팅 (AVAudioSession vs LiveKit WebRTC 충돌)
   - ② 음향 에코 루프 차단 (VAD + 500ms 가드, interruption 비활성)
   - ③ 음성-자막 모달리티 동기화 (AudioTranscriptionConfig 주입)
   - ④ 0ms 입력 언어감지 (유니코드 블록 분석기)
5. **아키텍처** — iPhone → LiveKit Cloud → Python Agent 3단 흐름도
6. **데모** — 영상 자리(플레이스홀더) + CSS 폰 목업
7. **시행착오 포스트모템 5건** — progress.md 기반 (자동 구성 삽질, 에코 루프 변천사, 모델명 사고, OSStatus -50, 비동기 스트림 이중소비 등)
8. **기술 스택** — 4분류
9. **CTA / 푸터** — GitHub 링크

> 모든 기술 내용은 저장소의 `README.ko.md` + `progress.md`에서 추출한 **검증된 사실**입니다.

## 3. 사람이 채워야 하는 빈칸 (당신이 대신 채워줄 수 있으면 채워주세요)

### A. 본인/연락처 (Hero·푸터)
- 이름(한/영), 한 줄 소개, 이메일, 블로그/링크드인, 아바타 사용 여부

### B. 프로젝트 메타 (신뢰도)
- 개발 기간 / 진행 형태(개인·팀) / 본인 역할 / 개발 동기(2~3문장) / 실기기 테스트 환경

### C. 데모 자료 ⭐ (임팩트 최대)
- 데모 영상 링크(YouTube/Vimeo) 또는 mp4, 임베드 방식, 앱 스크린샷 1~4장, GIF(선택)

### D. 성과·수치 (선택)
- 추가/수정할 핵심 수치, 정량 결과, 수상·발표 이력

### E. 회고 (선택)
- 가장 크게 배운 점, 개선하고 싶은 점, 회고 섹션 추가 여부

## 4. 당신(외부 AI)에게 구체적으로 요청하는 것

다음 중 가능한 것을 답변으로 주세요:

1. **자료 보강** — 위 3번 항목 중 당신이 작업 맥락으로 알고 있는 값(개발 기간, 역할, 동기, 테스트 환경, 강조할 수치 등)을 채워서 표로 회신.
2. **사실 검증** — 2번 구성에 기술적으로 틀리거나 과장된 표현이 있으면 지적.
3. **콘텐츠 피드백** — 빠진 핵심 기능/난제, 더 부각할 셀링 포인트, 섹션 순서 제안.
4. **데모 연출 제안** — 영상/스크린샷을 어떻게 배치·캡션하면 좋을지.
5. **누락 위험** — 채용담당자 관점에서 이 페이지에 꼭 있어야 하는데 빠진 것.

> 회신은 "표 + 짧은 코멘트" 형식이면 그대로 페이지에 반영하기 좋습니다.
